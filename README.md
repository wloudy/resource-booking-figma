# Resource Booking Frontend

Frontend-приложение для организации и управления бронированием общих ресурсов:
конференц-залов, оборудования, специалистов и других внутренних ресурсов компании.

## Возможности

- Календарь занятости ресурсов по дате.
- Создание бронирований с выбором ресурса, даты и времени.
- Проверка конфликтов по времени (защита от двойного бронирования).
- CRUD управления ресурсами (создать, редактировать, удалить).
- Уведомления по событиям и переключатель синхронизации.
- Сохранение данных в `localStorage`.

## Маршруты (в стиле макета Figma)

- `/` — стартовый экран (вход/регистрация)
- `/app/people` — раздел "Люди"
- `/app/space` — раздел "Пространство"

Примечание: пункты `Аппаратура` и `Профиль` остаются в шапке как элементы интерфейса, но без отдельных страниц.

## Стек

- Vue 3
- Vue Router 4 (версия роутера, не версия Vue)
- Vuex 4 (версия state-менеджера, не версия Vue)
- Vuetify 4 (версия UI-библиотеки, не версия Vue)
- Vite

## Где вставлять свои ссылки на картинки

Все ссылки на изображения вынесены в файл:

- `src/data/imageLinks.js`

Важно: вставляй **прямые ссылки на файл картинки**, например:

- `https://site.com/image.jpg`
- `https://site.com/pic.png`
- `https://site.com/anim.gif`

Ссылки на страницы вида `https://tenor.com/.../view/...` не подходят.

Там можно заменить:

- `authBackground` — фон стартового экрана
- `topbarBackground` — фон верхней панели в приложении
- `spaces[]` — фото для карточек раздела "Пространство/Аппаратура"
- `people[]` — фото для карточек раздела "Люди"

## Запуск

```bash
npm install
npm run dev
```

Если на Windows есть локальная проблема с `ComSpec`:

```powershell
$env:ComSpec='C:\Windows\System32\cmd.exe'
npm install
npm run dev
```

Сборка production:

```bash
npm run build
```
